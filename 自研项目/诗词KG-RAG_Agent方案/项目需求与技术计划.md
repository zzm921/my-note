# 诗词 AI Agent 项目需求说明书

> 版本：v1.0
> 最后更新：2026-07-01

---

## 一、项目概述

### 1.1 项目背景

构建一个基于"一句话/一张图"即可从诗词库中找到最匹配诗词的 AI Agent。用户输入任意口语化表达或一张图片，系统通过语义理解、情感分析、场景匹配，返回意境最贴切的诗词及其完整信息。

### 1.2 项目目标

- **核心功能**：用户输入一句话或一张图，系统返回 Top-3 最匹配的诗词，附带匹配理由
- **数据维度**：支持情感、场景、意象、时间、地点、作者多维度匹配
- **展示信息**：诗词正文 + 作者 + 朝代 + 白话译文 + 创作背景 + 写作地点
- **交互方式**：飞书 Bot（Hermes 渠道）

### 1.3 用户场景

```
用户说: "今天加班到深夜，看到窗外的月亮，有点想家了"
系统返回:
  Top-1: 《静夜思》李白 — "举头望明月，低头思故乡"
  匹配理由: 深夜思乡场景，月意象高度匹配
  作者: 李白（701-762）
  写作时间: 727年
  写作地点: 安陆市（今湖北孝感）

用户上传一张夕阳湖面的照片:
系统返回:
  Top-1: "落霞与孤鹜齐飞，秋水共长天一色" — 王勃
```

## 二、功能需求

### 2.1 核心功能（P0）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **文本语义匹配** | 用户输入一句话，系统理解语义/情感/场景，匹配最相关的诗词 | P0 |
| **图片语义匹配** | 用户上传图片，系统识别场景/意象，匹配诗词 | P0 |
| **Top-3 结果返回** | 返回最匹配的 3 首诗词，含匹配理由和匹配度 | P0 |
| **诗词完整信息** | 正文、作者、朝代、译文、背景、写作地点、写作时间 | P0 |

### 2.2 增强功能（P1）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **多轮对话** | 用户可追问"还有类似的吗""是谁写的"，Agent 保持上下文 | P1 |
| **化用分析** | 用户问"这句化用了谁的诗"，调用搜韵 reference 接口 | P1 |
| **自动笺注** | 返回诗词时自动补充白话注释 | P1 |
| **用户反馈** | 点赞/点踩，用于优化排序 | P1 |

### 2.3 扩展功能（P2）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **按时间查诗** | "安史之乱期间写的诗" | P2 |
| **按地点查诗** | "在黄鹤楼写的诗" | P2 |
| **相似诗推荐** | "和这首意境相似的诗" | P2 |
| **创作辅助** | "帮我写一首七言绝句，主题是思乡" | P2 |

## 三、数据需求

### 3.1 数据来源

| 来源 | 数据内容 | 获取方式 |
|------|---------|---------|
| **搜韵 API** (open.cnkgraph.com) | 诗词正文、作者、时间、地点、韵部、笺注 | REST API 分批拉取 |
| **LLM 标注** (DeepSeek) | 情感标签、场景标签、意象标签、背景摘要 | 批量 LLM 标注 |
| **向量嵌入** (BGE/bge-large-zh-v1.5) | 诗词语义向量 | 本地生成 |

### 3.2 数据结构

```json
// 诗词主数据
{
  "id": "26460",
  "title": "静夜思",
  "author": "李白",
  "author_id": "15188",
  "dynasty": "盛唐",
  "content": "床前明月光，疑是地上霜。举头望明月，低头思故乡。",
  "translation": "..." ,                    // 搜韵 labelize 获取
  "background": "...",                     // LLM 摘要生成
  "writing_time": "727年",
  "writing_location": "安陆市",
  "location_id": "CN420982",
  "type": "绝句",
  "rhyme": "阳",
  "tags": {
    "emotions": ["思念", "孤独"],          // LLM 标注
    "scenes": ["深夜", "思乡"],            // LLM 标注
    "imageries": ["月", "霜", "床"]        // LLM 标注
  },
  "embedding": [0.123, 0.456, ...]         // BGE 向量
}
```

## 四、技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        接入层                                │
│           Hermes → 飞书 Bot / API                           │
└────────────────────────┬────────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────────┐
│                      编排层                                  │
│          RAGFlow / 自定义服务                                │
│          意图识别 → 路由 → 多路检索 → 生成回答              │
└────────────────────────┬────────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────────┐
│                      检索引擎                                │
│                                                             │
│  ┌─────────────────┐  ┌──────────────────┐  ┌───────────┐  │
│  │ 知识图谱 (Neo4j) │  │ 向量库 (Milvus)   │  │ 搜韵 API  │  │
│  │ 诗-人-地-时关系  │  │ 语义向量索引      │  │ 补充笺注  │  │
│  │ 情感-场景-意象   │  │ 相似度搜索        │  │ 化用分析  │  │
│  └────────┬────────┘  └────────┬─────────┘  └─────┬─────┘  │
│           └───────────┬────────┘                  │        │
│                   RRF 融合排序                     │        │
└────────────────────────┬────────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────────┐
│                      数据层                                  │
│                                                             │
│  ┌──────────────────┐  ┌────────────────┐                   │
│  │ PostgreSQL       │  │ 搜韵 API 源    │                   │
│  │ 诗词主数据       │  │ 原始数据       │                   │
│  │ 用户反馈         │  │ 增量同步       │                   │
│  │ 对话记录         │  │                │                   │
│  └──────────────────┘  └────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 技术栈

| 组件 | 选型 | 用途 |
|------|------|------|
| **图数据库** | Neo4j (Docker) | 知识图谱存储，多跳关系查询 |
| **向量库** | Milvus (Docker) | 诗词语义向量索引 |
| **关系型** | PostgreSQL / SQLite | 业务数据、用户反馈 |
| **LLM** | DeepSeek (已有) | 标签标注、意图识别、回答生成 |
| **嵌入模型** | BAAI/bge-large-zh-v1.5 | 中文语义向量生成 |
| **编排框架** | LangChain / LlamaIndex | Agent 工具链编排 |
| **后端框架** | Python FastAPI | API 服务 |
| **数据源** | 搜韵 API (open.cnkgraph.com) | 诗词数据获取 |
| **渠道** | Hermes → 飞书 | 用户交互入口 |

## 五、项目技术拆分

### 模块一：数据采集层

```
poetry-agent/data/
├── souyun_client.py          # 搜韵 API 封装（人员/作品/地点/工具）
├── batch_importer.py         # 按朝代分批拉取全量数据
├── schema_mapper.py          # 搜韵字段 → 本地字段映射
└── data_cleaner.py           # 时间格式归一化、地点ID兼容
```

### 模块二：知识图谱层

```
poetry-agent/graph/
├── neo4j_schema.py           # 节点/关系定义（Cypher DDL）
├── neo4j_importer.py         # 数据导入 Neo4j
├── cypher_templates.py       # 预定义 Cypher 查询模板
└── graph_retriever.py        # 图谱检索器（多跳查询）
```

### 模块三：语义检索层

```
poetry-agent/vector/
├── embedding_service.py       # 向量生成服务（BGE 模型）
├── milvus_client.py           # Milvus 连接 & 索引管理
├── vector_retriever.py        # 向量检索器（语义匹配）
└── hybrid_search.py           # 多路检索融合（RRF）
```

### 模块四：标注层

```
poetry-agent/labeling/
├── emotion_labeler.py         # 情感标注（LLM）
├── scene_labeler.py           # 场景标注（LLM）
├── imagery_extractor.py       # 意象提取（LLM）
└── background_summarizer.py   # 背景摘要生成（LLM）
```

### 模块五：Agent 编排层

```
poetry-agent/agent/
├── intent_router.py           # 意图识别 + 路由
├── tools.py                   # 工具定义（3-4个核心工具）
├── memory.py                  # 多轮对话记忆
├── prompt_templates.py        # Prompt 模板管理
└── response_parser.py         # 回答格式化
```

### 模块六：API 服务层

```
poetry-agent/api/
├── main.py                    # FastAPI 入口
├── search_router.py           # 搜索接口
├── feedback_router.py         # 反馈接口
└── hermes_bridge.py           # Hermes 接入适配
```

## 六、编码先后顺序

### Phase 1：数据基础设施（Day 1-3）

```
优先级: P0
目标: 能拉数据、能存数据、能搜数据

顺序:
  1. souyun_client.py         ← 先调通搜韵 API，确认数据结构
  2. neo4j_schema.py          ← 定义图谱 Schema
  3. embedding_service.py      ← 部署 BGE 模型，能生成向量
  4. batch_importer.py         ← 从搜韵拉数据并导入 Neo4j + 生成向量
  5. data_cleaner.py           ← 时间/地点字段归一化

验证标准: 
  Neo4j 里有 1000+ 首诗词，能跑通一条 Cypher 查询
  Milvus 里有向量索引，能跑通一条语义搜索
```

### Phase 2：核心检索链路（Day 4-6）

```
优先级: P0
目标: 文本输入 → 找到匹配诗词

顺序:
  1. cypher_templates.py       ← 写 3-4 个核心 Cypher 模板
  2. graph_retriever.py        ← 图谱检索器
  3. vector_retriever.py       ← 向量检索器  
  4. hybrid_search.py          ← 多路检索融合（RRF）
  5. emotion_labeler.py        ← 情感标注（批量跑）
  6. scene_labeler.py          ← 场景标注（批量跑）
  7. imagery_extractor.py      ← 意象提取（批量跑）

验证标准:
  输入"加班到深夜好累" → 返回意境匹配的诗词 Top-3
```

### Phase 3：Agent 编排（Day 7-8）

```
优先级: P0
目标: 用户能通过飞书 Bot 对话

顺序:
  1. intent_router.py          ← 意图识别
  2. tools.py                  ← 3 个核心工具
  3. prompt_templates.py       ← 回答生成 Prompt
  4. response_parser.py        ← 格式化输出
  5. hermes_bridge.py          ← 接入飞书
  6. memory.py                 ← 多轮对话记忆

验证标准:
  飞书 Bot 上输入"加班好累" → 返回诗词 + 匹配理由
```

### Phase 4：增强功能（Day 9-10）

```
优先级: P1
目标: 笺注、化用、反馈

顺序:
  1. 接入搜韵 labelize 接口    ← 自动笺注
  2. 接入搜韵 reference 接口   ← 化用分析
  3. feedback_router.py        ← 用户反馈链路
  4. 图片输入（Chinese-CLIP）   ← 多模态

验证标准:
  结果附带白话注释
  追问"这句化用了谁的诗"能回答
  用户反馈能入库
```

### Phase 5：优化迭代（持续）

```
优先级: P2
目标: 完善体验

  1. 多维度过滤（按时间/地点/朝代）
  2. 相似诗推荐
  3. 排序权重调优（基于反馈数据）
  4. 全量数据导入（200万首）
  5. 创作辅助功能
```
